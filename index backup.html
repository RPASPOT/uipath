    🔰3. State Machine

    • ✅State Machine is a type of Workflow. It provides a modelling style with which you can model your workflow in an event-driven manner. It goes into a state when it is triggered by an activity, and it exits that state when another activity is triggered.
    • ✅The State activity contains three sections, Entry, Exit and Transition(s) ✅Entry Section: It specified action that occures when this state is trasition to. When workflow instance enters a state, any activities in the entry action execute. ✅Exit Section: Any activity exetuted when existing the state. When a transation to another state is confirmed the activities in the exit action are executed, even if the state transation back to the same state. ✅Transition Section: A trasition represents the transation between twi states. It displays all the transations linked to the selected state. It is used to add conditions based on which to jump from on state to another.
    • ✅Transitions Designer: Transition are created by dragging a line from one state to another. It is expanded when you double-click them. It has various sections: ✅Source- It indicates the state from which this transation originates. ✅Trigger- Typically the Trigger is an activity that waits for some type of event to occur, but it can be any activity, or no activity at all. A Transition's Trigger is scheduled when the transation's source state's Entry action is completed. ✅Condition- This section is used to apply condition. Once the trigger activity completes, the Condition is evaluated. If there is no Trigger activity then the Condition is immediately evaluated. ✅Action- Specifies the activity that is executed when the trigger activity completed and the Condition evaluates to true. ✅Destination- It indicate the state that the state machine transations to after the transation on completes.
    • ✅When Project has certain states and Switches during execution.
    • ✅Uses a finite number of states in its execution.
    • ✅It can go into a state when it is triggered by an activity, and it exists that state when another activity is triggered.
    • ✅State and Final State are specified activity.
    • ✅There are two activities that are specific to state machines, namely State and Final State, found under Workflow > State Machine.

    🔰4 .Global Exception Handler

    • ✅Global Exception Handler is a type of workflow which is used to handle the errors encountered in Project. When error occurs, It redirects to the Global Handler workflow to log the error and handle the errors.
    • ✅Global Exception Handler needs to define once in the Project, Itwill execute every time an activity fails to execute.
    • ✅It helps you to execute a workflow or Activity every time an execution error occurs, You can choose option to Ignore the exception, Retry, Break or Continue the execution.
    • ✅When you add a Global Handler Workflow, It adds all required activities, Variable and Arguments as a template. Do not Delete the Variable and Arguments from the Global Handler Workflow template.
    • ✅There are few option can be used in UiPath Studio to handle the errors. Abort: It stops the execution after running the current handler. Ignore: It ignores the exception and continue the execution from the next activity. Retry: It will retry execution the current activity which is faulted. Continue: It runs the Global Exception Handler workflow.

    Example of How to Use a Global Exception Handler

    • ✅
    • ✅The entry point of a process (Usually Main.xaml) cannot be declared as an exception handler.
    • ✅
    • ✅If Project,Json file has "exceptionHandlerWorkflow": :GlobalHandler.xaml", It means your Project has Global Exception Handler applied, Workflow name is changed automatically in Project.Jsonwhen you changed the Global Handler workflow name.

    🔰1. What is variable?

    • ✅Variables are storage spaces that can be used to hold multiple data entries( values) of the same data type. In Uipath Variables are used to store values that they can use later or across their automation Process. All variables are unique which means you cannot create two variables of the same name. For Example:
    • ✅EmailAddress can be a variable that holds the value “sample@rpatutorials.com”. like these, variables can be used to store multiple values such as Name, Age and etc.
    • ✅The value of each variable can be changed through an external input, data manipulation, or passing from one exertion to another and each variable is unique
